Social Helper provides various functionalities such as: user donation, donation management and resource request and distribution. The application consist of two major components: one that manages the donations and one that manages how they are allocated. The aim of the application will be to make as easy as possible for its users to keep track of everything electornically and make them more effective such that they can help more people.

  • The user donation page allows the user to find the nearest warehouse to send his items. He also has an option to request transportation handling by the charity which may be free depending on the value of the donation
  • After the donation has arrived a warehouse employee will weigh the package and analyze its content. He will then proceed to add the donation to the database with all its necessary attributes.
  • The server will then calculate if the donator needs to be charged any shipping fee and inform him of it
  • An authorized charity employee can create a request for resources base on an event that happened in the world such as droughts, floods, accidents, etc.
  • Only someone with enought privileges will be able to approve such a request (either partially or fully, depending on the resources available).

The application will go through multiple stages of development and the progress bar bellow will show the progress made so far.
