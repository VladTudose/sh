Requests

GET /requests

Lists the current requests for resources in the system
Parameters: none
Response: 200, the ids of all the requests in the system | 403, if the user is not authenticated or authorized

POST /request

Creates a new request based on the given parameters which describe an accident or a disaster
Parameters: location, number of victims, additional information, list of resources required
Response: 200, OK, returns the id of the created request | 403, if the user is not authenticated or authorized

POST /request/{id}/approve/full

Approve the request with the given id only if we have enough resources
Parameters: none
Response: 200, OK | 406, if we don't have enough resources | 403, if the user is not authenticated or authorized

POST /request/{id}/approve/partial

Approve the request with the given id; if we don't have enough resources we will only sent what we have
Parameters: none
Response: 200, OK | 403, if the user is not authenticated or authorized

GET /request/{id}/deny

Deny the request with the given id (eventually with an explication)
Parameters: none
Response: 200, OK | 403, if the user is not authenticated or authorized

GET /request/{id}/coverable

Check if we have enough resources to fully satisfy the request with the given id
Parameters: none
Response: 200, we have enough resourced | 406, if we don't have enough resources | 403, if the user is not authenticated or authorized

Users

POST /login/{email}

Login the user with the given email
Parameters: none
Response: 200, sets a cookie for authentication

POST /logout

Logout the user with the given email
Parameters: none
Response: 200, removes the cookie for authentication

Categories

GET /categories/root

Get the names of the roots of each taxonomy in the application (product taxonomy, events taxonomy etc.)
Parameters: none
Response: 200, the name of the roots of each taxonomy in the application | 403, if the user is not authenticated or authorized

GET /category/{name}/subcategories

Get the names of the subcategories for the given category
Parameters: none
Response: 200, the names of the subcategories for the given category | 403, if the user is not authenticated or authorized

GET /category/{name}/attributes

Get the names of the attributes for the given category
Parameters: none
Response: 200, the names of the attributes for the given category | 403, if the user is not authenticated or authorized

Resource

POST /resource/

Adds a new resource with the given parameters
Parameters: category, estimated value, attributes
Response: 200, OK, returns the id of the created resource | 403, if the user is not authenticated or authorized

GET /resource/{id}

Get the resource with the given id
Parameters: none
Response: 200, OK, returns the resource with the given id | 403, if the user is not authenticated or authorized

Donations

GET /donations

Get the ids for all donations
Parameters: none
Response: 200, OK, returns the ids of all the donations in the system | 403, if the user is not authenticated or authorized

GET /donation/{id}

Get the donation with the given id
Parameters: none
Response: 200, OK, returns the donation with the given id | 403, if the user is not authenticated or authorized

POST /donation

Create a new donation with the given parameters. It also calculates how much the person who donated has to pay (may be zero if the cost of the donated items exceeds the cost of pick-up and transport) and charge him accordingly.
Parameters: pick-up location, drop-out location, weight, items, credit card information
Response: 200, Created | 403, if the user is not authenticated or authorized

Warehouses

GET /warehouses

Get the ids for all the warehouses within a given radius from a given location
Parameters: location, radius
Response: 200, OK, returns the ids of warehouses within the given radius form the given location | 403, if the user is not authenticated or authorized

GET /warehouse/{id}

Get the informations for the warehouse with the given id (location, hours, etc.)
Parameters: none
Response: 200, OK, the informations for the warehouse with the given id | 403, if the user is not authenticated or authorized
